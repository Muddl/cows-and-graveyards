# Specification: Main Menu, Pause Menu & Trip Persistence

**Track ID:** menus_20260318
**Type:** Feature
**Created:** 2026-03-18
**Status:** Draft

## Summary

Add a main menu with slot-based trip management and a pause menu with
return-to-menu support. Both menus use a consistent UI design language,
properly isolate game input when active, and include a quit option on
non-mobile platforms only.

## Context

Cows & Graveyards currently drops the player directly into a live game
with no way to navigate back or manage multiple trips. Adding menus
completes the basic out-of-game experience and enables the trip persistence
system the product needs for repeated play sessions.

## User Story

As a player, I want more control over the game state via GUI elements
such that I have a better out-of-game experience.

## Acceptance Criteria

- [ ] A main menu is shown at launch with options: Start New Trip, Load
      Trip (up to 3 saved slots), and Quit (desktop only)
- [ ] Up to 3 trips are saved persistently across app runs; each slot
      stores its left/right scores and can be loaded or overwritten
- [ ] A pause menu is accessible during a game with options: Resume,
      Return to Main Menu, and Quit (desktop only)
- [ ] A consistent GUI design language (shared theme / style) is applied
      across all menus and existing HUD elements
- [ ] No game-state inputs (cow taps, graveyard buttons) are processed
      while any menu is open

## Dependencies

- Existing game loop: GameScene, GameState, ScoreTracker, GraveyardButton,
  ScoreHud — all from prior tracks

## Out of Scope

- High-fidelity visual assets — placeholders are acceptable
- Audio cues, animations, or transitions beyond functional UI
- Trip history, statistics, or any data beyond left/right scores per slot

## Technical Notes

- Platform detection via `OS.GetName()`: hide Quit on "Android" / "iOS"
- Trip persistence via Godot `FileAccess` writing JSON to `user://trips.json`;
  validate on load to handle corrupt/missing files
- Input isolation: `GameScene.SetProcessInput(false)` when any menu is
  open; re-enable on resume — test boundary carefully to prevent tap bleed
- Pause menu triggered by a persistent pause button in the CanvasLayer
  (MOUSE_FILTER_STOP, always on top) — same input-guard pattern as
  GraveyardButton
- Switching trips must save the current trip state before loading the new
  slot to avoid data loss
