# Specification: Graveyard Mechanic

**Track ID:** graveyard_20260318
**Type:** Feature
**Created:** 2026-03-18
**Status:** Draft

## Summary

Add a persistent graveyard trigger button on each side of the road. Tapping
it spawns a headstone placeholder that drives past the car on the same side,
and zeroes out the opposing side's score.

## Context

Cows & Graveyards is built around the classic road-trip cow-counting game.
The graveyard rule is central to that game: passing a graveyard on your side
wipes your own cows — here adapted so the *opposing* player's score is
zeroed. This is the second pillar of the core gameplay loop, building
directly on the tap, spawn, and score infrastructure from the initial game
scene track.

## User Story

As a player, I want to tap the graveyard button on either side of the road
so that a graveyard placeholder drives past the car on that side and the
opposing side's score is reset to zero.

## Acceptance Criteria

- [ ] A graveyard trigger button exists for both sides of the road and is
      responsive to window size
- [ ] Tapping a graveyard button spawns a headstone placeholder that
      animates past the car on the same side as the button tapped
- [ ] A graveyard event zeroes the opposing side's score and the HUD
      reflects the change immediately

## Technical Notes

- This is core gameplay — every new method and integration point requires a
  test written before implementation (strict TDD)
- ScoreTracker needs targeted zero methods (ZeroLeft / ZeroRight) rather
  than full Reset, to preserve the unaffected side's score
- GraveyardButton is a UI Control anchored responsively; it must not
  interfere with existing cow tap detection (MOUSE_FILTER_STOP on the
  button rect only, pass-through elsewhere)
- GraveyardEntity drive-past behaviour is identical to CowEntity — same
  lerp animation, same lifecycle — but is a distinct scene and class for
  clear visual separation
